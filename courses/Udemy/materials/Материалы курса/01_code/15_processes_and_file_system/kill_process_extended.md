
# Завершение процессов и их дочерних процессов

Эта статья - результат вопросов.

Очень часто возникает ситуация:
Вы завершаете процесс (например VS Code), но часть его дочерних процессов продолжает работать.

Это происходит потому, что:

- обычное завершение через terminate или kill методы работает только с одним процессом
- дочерние процессы могут жить отдельно потом

Поэтому нужно уметь завершать **всё дерево процессов**, а не только один PID. Это реализуется
через системные утилиты ОС. Это консольные программы в самой ОС, и их можно вызывать как мы
смотрели, точно так же через скрипт, передавая им что мы хотим сделать.
**Делаю обзорную статью на ВСЕ 3 ОС:**

# Уровни завершения процессов

Можно условно выделить три уровня:

1. Мягкое завершение  
   process.terminate()  
   Просит процесс завершиться корректно

2. Жесткое завершение  
   process.kill()  
   Завершает процесс принудительно

3. Завершение дерева процессов  
   Использование системных утилит или групп процессов  
   Завершает процесс и все его дочерние процессы

---
---
Пару заметок перед началом:

* Внимание! Имейте ввиду, что если вы что-то запускаете через subprocess
с флагом shell=True, то PID процесса в объекте процесса который вернет Popen
будет PID оболочки, которая уже могла закрытся, если программа через эту
оболоку запускалась в режиме отделения от обоочки (например 
через start на Windows или open/xdg-open на Mac/Linux).
Поэтому чтоб иметь у себя PID реального процесса, запускайте исполняемый файл
программ напрямую через Popen без shell=True, если это возможно.

* И ещё один NOTE перед началом:
Popen возвращает объект описывающий процесс - это мы знаем. Но хочу подсказать,
что мы можем хранить этот объект в списке, если это будет удобно, а не просто
на месте из него методы использовать. Это позволит потом, из списка забрать объект
процесса, и сделать что-то  с ним позже:
```python
task_list = []

process = subprocess.Popen([program])

task_list.append(process)
```

# Windows

## Завершение по PID (с дочерними процессами)

Используется консольная утилита taskkill с соответствующими флагами:

```python
import subprocess

subprocess.run([
    "taskkill", "/PID", str(job.pid), "/T", "/F"
])
```
Пояснение:

* /PID - указание процесса по ID
* /T - завершить дерево процессов
* /F - принудительное завершение

Это более надежный способ, чем terminate или kill.

---

## Завершение по имени процесса

```python
subprocess.run([
    "taskkill", "/IM", job.name, "/T", "/F"
])
```

Примеры имен:

* python.exe
* chrome.exe
* code.exe

Пояснение:

* /IM - имя процесса
* удобно, если PID уже неизвестен, но завершит все процессы с таким именем, даже если это не "ваш".

# Linux

## Вариант 1. Завершение через pkill и kill

Используется консольная утилита pkill и kill с соответствующими флагами:

```python
import subprocess

# Сначала убиваем дочерние процессы
subprocess.run([
    "pkill", "-TERM", "-P", str(job.pid)
])

# Потом родительский
subprocess.run([
    "kill", "-TERM", str(job.pid)
])
```

Пояснение:

* -P - означает "дочерние процессы"
* сначала завершаем детей, потом родителя

---

## Вариант 2. Группа процессов (рекомендуется)

Если процесс был запущен так:

```python
subprocess.Popen(..., start_new_session=True)
```

То можно завершить всю группу:

```python
import os
import signal

os.killpg(job.pid, signal.SIGTERM)
```

Пояснение:

* создается отдельная группа процессов
* job.pid становится ID группы
* killpg завершает всю группу сразу (kill process group функция из os)
* signal.SIGTERM - это просто номер сигнала на процесс (в линукс их часто используют)

Это самый правильный способ в Linux и Mac системах.

---

## Вариант 3. По имени процесса

```python
subprocess.run([
    "pkill", "-f", job.name
])
```

Пояснение:

* -f ищет по всей командной строке
* удобно, но опасно, так как может завершить другой процесс просто с тем же именем.

# Универсальный кроссплатформенный вариант

```python
import os
import sys
import signal
import subprocess


# Завершение по PID:
if sys.platform == "win32":
    subprocess.run([
        "taskkill", "/PID", str(pid), "/T", "/F"
    ])
else:
    os.killpg(pid, signal.SIGTERM)

# Завершение по имени:
if sys.platform == "win32":
    subprocess.run([
        "taskkill", "/IM", name, "/T", "/F"
    ])
else:
    subprocess.run([
        "pkill", "-f", name
    ])
```

# Важные нюансы

## 1. SIGTERM и SIGKILL

* SIGTERM - "попробуй корректно завершиться"
* SIGKILL - "завершись немедленно"

SIGKILL не дает процессу сохранить данные. Это Linux и Mac только.

---

## 2. Потеря данных

Принудительное завершение может привести к:

* потере несохраненных файлов
* потере состояния программы (в файл что-то не записалось/не сохранилось)
* повреждению данных (в файл что-то не записалось/не сохранилось)

---

## 3. Завершение по имени опасно

Пример:

```bash
pkill -f python
```

Это завершит ВСЕ процессы Python.

---

## 4. Почему killpg может не работать (не Windows)

Если процесс запущен без:

```python
start_new_session=True
```

то:

* он не является лидером группы
* killpg может не сработать

---

# Краткое резюме:

Для надежного завершения процессов:

* Windows - используйте taskkill с /T
* Linux и macOS - используйте killpg или pkill

Совет:
* Не стесняйтесб подгугливать моменты, на тему взаимодейтсвия с ОС. Мы уже умеем
из программы запускать процессы через run и Popen, а многое можно делать через это И
утилиты самой ОС.
