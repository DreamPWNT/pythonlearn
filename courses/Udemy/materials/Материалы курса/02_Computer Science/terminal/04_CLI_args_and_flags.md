## Аргументы CLI программ

- Аргументы программы это всё, но есть образная классификация:
    - просто позиционные значения
    - флаги
    - флаги со значениями (как параметр и аргумент)

## Основные непонятные понятия:

Флаг - какая-то конретная опция или парметр, это фактически одно и тоже.
Может использоваться просто как факт ее наличия (передачи). Фактически при его наличии
это знаит True.
Или служить четким маркером (как параметр) для которого передается значение следующее за ним.

Пример флагов, которые просто сами по себе, как аргументы, что-то значат для процесса:
```bash
python -u app.py  # флаг -u (без буферизации)
python -i app.py  # флаг -i (интерактивно)
```
Пример флагов, которые выступают в роли "как-бы" параметров, после которых передается значение:
```bash
ping -c 5 google.com  # флаг -c (как параметр для количества 5)
# или на windows
ping -n 5 google.com
ping google.com -t
```
## Базовые правила хорошего тона (де-факто стандарты UNIX):
*(нет обязанности им следовать, но было бы не плохо, ведь всем приятно, когда всё предсказуемо, правда?)*

- Аргументы и флаги разделяются пробелами.
- Короткие флаги. Однобуквенные опции, обычно начинаются с одного дефиса (например, -a, -v).
- Длинные флаги: Полноценные слова, начинающиеся с двух дефисов (например, --all, --verbose).
- Корооткие и длинные флаги часто дублируются. Если есть -h, то есть и --help.
- Флагам могут требоваться значения. Например, в команде some_command --output filename.txt, filename.txt является значением для флага --output ( -o короткая альтернатива для --output).
- Если для флага требуется значение, то можно использовать или пробел, как в пункте выше, или символ =, вот так --output=filename.txt
- Порядок аргументов (и флагов в том числе) может иметь значение. Например, в команде copy file1.txt file2.txt, file1.txt и file2.txt являются позиционными аргументами и порядок их следования определяет, какой файл является исходным, а какой целевым.
- Если есть возможность, хорошим тоном является возможность передачи коротких флагов "слитно", если их комбинация допускатеся -la.
- Если короткий флаг принимает значение, оно относится к последнему флагу: -xvf file.tar.
- Длинные флаги не объединяются.
- "Голый" -- без имени флага, завершает обработку флагов и всё после него считается позиционными аргументами.
- Тем не менее синтаксис и набор доступных аргументов и флагов зависит от конкретной программы. Но желательно следовать общим правилам (по возможности).


Пример реального:
```bash
python -um -Xu8 package
```
- -u - unbuffered I/O
- -m - запуск как модуля
- -X - спец-опции интерпретатора Python
- u8 - значение для -X (принудительное utf8)
- package - тоже позиционный аргумент
