# Руководство по переменным и базовому синтаксису в `sh`, `bash` и `zsh`

Это руководство поможет вам освоить базовый синтаксис оболочек `sh`, `bash` и `zsh`. В первую очерь это нужно просто для базового знакомства, чтоб знать что такое есть. Эта информация не предназначена для заучивания в рамках этого курса, как в принципе и вообще. Когда вам это будет необходимо использовать, эта информация поможет быстро сориентироваться и моменты сами запомнятся "на ходу".

## 1. Переменные

### 1.1 Объявление переменных

Переменные в shell создаются простым присваиванием значения. Пробелы вокруг знака `=` недопустимы, так как это вызовет синтаксическую ошибку. Все значения в shell по умолчанию интерпретируются как строки, даже если они выглядят как числа.

```bash
name=Alex     # ВАЖНО: без пробелов вокруг "="
age=30        # По умолчанию - строки
```

### 1.2 Чтение значения переменной

Для доступа к значению переменной используется символ `$` перед её именем:

```bash
echo $name    # Выводит: Alex
```

Пример использования переменной в команде:
```bash
name="Alex"
echo "Hello, $name!"  # Выводит: Hello, Alex!
```

- Shell-переменные создаются в момент присваивания.
- Пробелы вокруг `=` вызовут ошибку.
- Все значения - строки (в отличие от Python, где есть явные типы данных).


Желательно использовать фигурные скобки `${}` для явного указания границ имени переменной, особенно если за переменной следует текст:

```bash
echo "${name}123"   # Выводит: Alex123
```

Без фигурных скобок интерпретатор оболочки может попытаться найти переменную `name123`, что приведёт к ошибке или пустому выводу:

```bash
var=hello
echo $varworld  # Ошибка: ищет переменную varworld
echo ${var}world  # Выводит: helloworld
```

### 1.3 Изменение значения

Чтобы изменить значение переменной, просто присвойте ей новое значение:

```bash
name="Sasha"
```

### 1.4 Добавление к существующему значению

Для добавления текста к существующему значению переменной можно использовать конкатенацию:

```bash
name="${name} Tukas"
```

В `bash` и `zsh` также доступен оператор `+=`:

```bash
name+=" Tukas"
```

> **Примечание**: Оператор `+=` не поддерживается в `sh`, используйте конкатенацию с `${}` для совместимости.

## 2. Переменные окружения

Переменные окружения - это переменные, которые доступны дочерним процессам (например, другим скриптам или программам, таким как `python` или `ls`). По своему поведению никак не отличаются от обычных переменных. Чтобы сделать переменную переменной окружения, используется команда `export`:

```bash
export name="Alex"
```

Или задайте переменную и экспортируйте её отдельно:

```bash
name="Alex"
export name
```

Пример:
```bash
export name="Alex"
env | grep name  # Показывает name=Alex в списке переменных окружения
bash -c 'echo $name'  # Выводит: Alex
```

> **Важно**: Только экспортированные переменные наследуются дочерними процессами. Именно этот факт делает их переменными окружения.

## 3. Арифметика

Shell по умолчанию рассматривает все переменные как строки, но для выполнения арифметических операций можно использовать конструкцию `$(( ))`. Символ `$` значит какое-то выражение, а двойные круглые скобки значат что действия проводить как над числами:

```bash
count=0
count=$((count + 1))  # Отдельно для count в скобках $ не нужно
echo "$count"   # Выводит: 1
```


В `bash` и `zsh` (кроме `sh`) также доступна конструкция `(( ))` для арифметики:

```bash
((count++))
```

В `sh`!!! для арифметики часто используется команда `expr`
(expression - выражение):

```bash
count=0
count=$(expr $count + 1)
echo $count  # Выводит: 1
```

> **Примечание**: Конструкции `$(( ))` и `(( ))` предпочтительнее в `bash/zsh`, так как они более современные и удобные.

## 4. Условия и сравнения

### 4.1 Строковые сравнения

Для проверки условий в shell используется команда `test` (или её синоним `[ ]`). Строковые сравнения выполняются с помощью операторов `=` (равно) и `!=` (не равно):

> Синтаксис условий думаю будет понятен.

```bash
if [ "$name" = "Alex" ]; then
  echo "Hello, Alex!"
fi
```

> **Важно**: Всегда заключайте переменные в кавычки (`"$name"`) в `[ ]`, чтобы избежать ошибок, если переменная пуста.

В `bash` и `zsh` (кроме `sh`) доступен расширенный синтаксис `[[ ]]`, который более гибкий и поддерживает шаблоны:

```bash
if [[ $name == Alex ]]; then
  echo "Hello again!"
fi
```

> **Примечание**: `[[ ]]` не требует экранирования специальных символов и поддерживает шаблоны, например, `A*` для строк, начинающихся с "A". Этот синтаксис недоступен в `sh`.

### 4.2 Числовые сравнения

Для числовых сравнений используются операторы `-eq`, `-ne`, `-gt`, `-lt`, `-ge`, `-le`:
> **!!!** Легко запомнить так как созвучно с методыми сравнения в python.

```bash
if [ "$age" -gt 18 ]; then
  echo "Adult"
fi
```

**Операторы сравнения:**
- `-eq` — равно
- `-ne` — не равно
- `-gt` — больше
- `-lt` — меньше
- `-ge` — больше или равно
- `-le` — меньше или равно

Пример с использованием переменных:
```bash
a=10
b=20
if [ $a -lt $b ]; then
  echo "$a is less than $b"
fi
```

Пример с `if-elif-else`:
> **Примечание**: Пример объясняющий получение ввода от пользователя в переменную обоочки будет ниже.
```bash
#!/bin/bash
echo "Please enter a number: "
read num
if [ $num -gt 0 ]; then
  echo "$num is positive"
elif [ $num -lt 0 ]; then
  echo "$num is negative"
else
  echo "$num is zero"
fi
```

## 5. Удаление переменных

Для удаления переменной используется команда `unset`:

```bash
unset name
```

После этого переменная становится недоступной, и попытка вывести её значение (`echo $name`) ничего не вернёт.

## 6. Обработка аргументов командной строки

Shell предоставляет специальные переменные для работы с аргументами командной строки (это когда запускается файл скрипт оболочки с доп. аргументами):

```bash
echo $0      # Имя скрипта
echo $1      # Первый аргумент
echo $2      # Второй аргумент
echo $#      # Количество аргументов
echo "$@"    # Все аргументы списком
```

Пример скрипта:
```bash
#!/bin/bash
echo "Script name: $0"
echo "First argument: $1"
echo "Second argument: $2"
echo "Number of arguments: $#"
echo "All arguments: $@"
```

Если выполнить `./script.sh arg1 arg2`, вывод будет:
```
Script name: ./script.sh
First argument: arg1
Second argument: arg2
Number of arguments: 2
All arguments: arg1 arg2
```

## 7. Ввод пользователя

Команда `read` позволяет считывать ввод пользователя и сохранять его в переменную:

```bash
read user_name
echo "Привет, $user_name!"
```

Можно считывать несколько значений за раз:
```bash
read first_name last_name
echo "Hello, $first_name $last_name!"
```

Если пользователь введёт "John Red", скрипт выведет "Hello, John Red!".

## 8. Значения по умолчанию

Для обработки случаев, когда переменная не установлена, используются конструкции `${var:-default}` и `${var:=default}`:

```bash
echo ${name:-"Guest"}   # Если переменная не установлена, выведет "Guest"
echo ${name:="Guest"}   # То же, но также присвоит "Guest" переменной name
```

Разница:
- `${var:-default}`: возвращает `default`, если `var` не установлена, но не изменяет `var`.
- `${var:=default}`: возвращает `default` и присваивает его `var`, если `var` не установлена.

## 10. Подсказки и рекомендации

- **Заключайте переменные в кавычки**: Используйте `"$var"`, чтобы избежать ошибок, если переменная содержит пробелы или пуста.
- **Не ставьте пробелы вокруг `=`**: Это вызовет синтаксическую ошибку при присваивании.
- **Используйте `[[ ... ]]` в `bash/zsh`**: Этот синтаксис более надёжен и гибок, чем `[ ]`.
- **Добавьте `set -e` в начало скрипта**: Это заставит скрипт завершаться при первой ошибке, что упрощает отладку.
- **Используйте `set -u`**: Это вызовет ошибку при попытке использовать неопределённую переменную, помогая избежать ошибок.

## 11. Функции (Functions)

Функции в `bash` и `zsh` позволяют группировать команды для повторного использования, делая скрипты более модульными и читаемыми.

### Определение функции

Есть два способа определения функции:

1. С использованием имени функции и круглых скобок:
   ```bash
   function_name () {
     # команды
   }
   ```

2. С использованием ключевого слова `function`:
   ```bash
   function function_name {
     # команды
   }
   ```

Первый способ более распространён.

### Вызов функции

Для вызова функции просто укажите её имя:
```bash
function_name
```

### Переменные в функциях

По умолчанию переменные в функциях глобальные, но можно сделать их локальными с помощью ключевого слова `local`:

```bash
my_function () {
  local var="local value"
  echo $var
}
my_function
echo $var  # Пусто, так как var локальна
```

### Возврат значений

Функции в `bash` не возвращают значения напрямую, но можно:
- Использовать `return` для возврата кода завершения (0 — успех, 1–255 — ошибка):

  ```bash
  my_function () {
    return 0
  }
  my_function
  echo $?  # Выводит: 0
  ```

- Вывести значение в стандартный вывод и захватывать его:
  ```bash
  my_function () {
    echo "Hello from function"
  }
  result=$(my_function)
  echo $result  # Выводит: Hello from function
  ```

### Передача аргументов

Аргументы передаются в функцию через `$1`, `$2` и т.д.:
```bash
greet () {
  echo "Hello, $1!"
}
greet "Alice"  # Выводит: Hello, Alice!
```

Также доступны `$#` (количество аргументов) и `$@` (все аргументы).

## 12. Массивы (Arrays) (только для `bash` и `zsh`)

Массивы позволяют хранить несколько значений в одной переменной. Они поддерживаются только в `bash` и `zsh`, но не в `sh`.

### Объявление массива

```bash
myArray=("cat" "dog" "mouse" "frog")
```

> **Примечание**: Элементы разделяются пробелами, а не запятыми. Строки заключаются в кавычки.

### Доступ к элементам массива

Для доступа ко всем элементам:
```bash
for str in "${myArray[@]}"; do
  echo $str
done
```

Для доступа к конкретному элементу (индексы начинаются с 0):
```bash
echo ${myArray[0]}  # Выводит: cat
echo ${myArray[1]}  # Выводит: dog
```

### Цикл по индексам массива

```bash
for i in "${!myArray[@]}"; do
  echo "element $i is ${myArray[$i]}"
done
```

## 13. Циклы for

Циклы `for` используются для итерации по списку элементов.

### Цикл for по списку

```bash
for i in 1 2 3 4 5; do
  echo "Number: $i"
done
```

### Цикл for по файлам

```bash
for file in *.txt; do
  echo "Processing $file"
done
```

### Цикл for с диапазоном (в `bash`)

```bash
for i in {1..5}; do
  echo $i
done
```

## 14. Работа с файлами

### Проверка существования файла

```bash
if [ -f "file.txt" ]; then
  echo "file.txt exists"
else
  echo "file.txt does not exist"
fi
```

### Чтение из файла

```bash
while read line; do
  echo $line
done < input.txt
```

### Запись в файл

```bash
echo "Hello, World!" > output.txt
```

### Добавление в файл

```bash
echo "More text" >> output.txt
```
