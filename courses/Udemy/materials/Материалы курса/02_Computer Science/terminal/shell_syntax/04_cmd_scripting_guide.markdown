# Руководство по переменным и базовому синтаксису в `cmd`

Это руководство поможет вам освоить базовый синтаксис командной оболочки `cmd` в Windows. В первую очерь это нужно просто для базового знакомства, чтоб знать что такое есть. Эта информация не предназначена для заучивания в рамках этого курса, как в принципе и вообще. Когда вам это будет необходимо использовать, эта информация поможет быстро сориентироваться и моменты сами запомнятся "на ходу".

---

## 1. Переменные

> **Важно**: В cmd нет разделения переменных на просто переменные и переменные окружения. Все переменные в cmd являются переменными окружения!!!

### 1.1 Объявление переменных

Переменные в `cmd` создаются с помощью команды `SET`. Пробелы вокруг знака `=` недопустимы, так как это вызовет синтаксическую ошибку. Все значения по умолчанию интерпретируются как строки.

```cmd
SET name=Alex
SET age=30
```
> **!!!**: `REM` это "символ" комментария в файлах сценариев для cmd. Это и другие ключевые слова можно писать в нижнем регистре, включая SET set, ECHO echo и тд.

Пример использования переменной в команде:
```cmd
SET name=Alex
ECHO Привет, %name%!  REM Выводит: Привет, Alex!
```

- Переменные создаются в момент присваивания.
- Пробелы вокруг `=` вызовут ошибку.
- Все значения - строки (в отличие от языков, таких как Python, где есть явные типы данных).

### 1.2 Чтение значения переменной

Для доступа к значению переменной используются символы `%` перед и после её имени:

```cmd
ECHO %name%  REM Выводит: Alex
```

> **Важно**: Если переменная не определена, `ECHO %name%` выведет `%name%`, если не использовать кавычки или проверку.

### 1.3 Изменение значения

Чтобы изменить значение переменной, просто присвойте ей новое значение:

```cmd
SET name=Sasha
```

### 1.4 Добавление к существующему значению

Для добавления текста к существующему значению переменной используется конкатенация:

```cmd
SET name=%name% Tukas
```

> **Примечание**: В `cmd` нет оператора `+=` для строк, как в `bash/zsh`. Используйте `%name%` для конкатенации.

## 2. Арифметика

В `cmd` арифметические операции выполняются с помощью команды `SET /A`. Это позволяет интерпретировать значения переменных как числа:

```cmd
SET /A count=0
SET /A count+=1
ECHO %count%  REM Выводит: 1
```

Поддерживаемые операции:
- `+` — сложение
- `-` — вычитание
- `*` — умножение
- `/` — деление
- `%` — остаток от деления

Пример:
```cmd
SET /A result=5*3
ECHO %result%  REM Выводит: 15
```

> **Примечание**: `SET /A` не поддерживает числа с плавающей точкой, только целые.

## 3. Условия и сравнения

### 3.1 Строковые сравнения
> Думаю что синтаксис условий уже будет интуитивно понятен.

Для проверки условий используется команда `IF`. Строковые сравнения выполняются с помощью оператора `==`:

```cmd
IF "%name%"=="Alex" ECHO Привет, Alex!
```

> **Важно**: Всегда заключайте переменные в кавычки `"%name%"`, чтобы избежать ошибок, если переменная пуста.

Пример с `IF-ELSE`:
```cmd
IF "%name%"=="Alex" (
    ECHO Привет, Alex!
) ELSE (
    ECHO Вы не Alex.
)
```

### 3.2 Числовые сравнения

Для числовых сравнений используются операторы `EQU`, `NEQ`, `LSS`, `LEQ`, `GTR`, `GEQ`:

```cmd
IF %age% GTR 18 ECHO Взрослый
```

**Операторы сравнения:**
| Оператор | Описание              |
|----------|-----------------------|
| EQU      | Равно                |
| NEQ      | Не равно             |
| LSS      | Меньше               |
| LEQ      | Меньше или равно     |
| GTR      | Больше               |
| GEQ      | Больше или равно     |

Пример с числовым сравнением:
```cmd
SET age=20
IF %age% GTR 18 (
    ECHO Взрослый
) ELSE (
    ECHO Несовершеннолетний
)
```

## 4. Удаление переменных

Для удаления переменной достаточно присвоить ей пустое значение:

```cmd
SET name=
```

После этого переменная становится неопределённой, и попытка вывести её значение `ECHO %name%` покажет `%name%`.

---

## 5. Обработка аргументов командной строки

`cmd` предоставляет специальные переменные для работы с аргументами командной строки. Это когда запускается файл скрипт сценария cmd с заранее записанными командами:

```cmd
ECHO %0  REM Имя скрипта
ECHO %1  REM Первый аргумент
ECHO %2  REM Второй аргумент
ECHO %*  REM Все аргументы
```

Пример скрипта:
```cmd
@ECHO OFF
ECHO Имя скрипта: %0
ECHO Первый аргумент: %1
ECHO Второй аргумент: %2
ECHO Все аргументы: %*
```

Если выполнить `script.bat arg1 arg2`, вывод будет:
```
Имя скрипта: script.bat
Первый аргумент: arg1
Второй аргумент: arg2
Все аргументы: arg1 arg2
```

## 6. Ввод пользователя

Команда `SET /P` позволяет считывать ввод пользователя и сохранять его в переменную:

```cmd
SET /P user_name=Введите ваше имя: 
ECHO Привет, %user_name%!
```

> **Примечание**: Ввод пользователя сохраняется как строка, и пробелы в вводе обрабатываются корректно.

---

## 7. Значения по умолчанию

В `cmd` нет прямого аналога конструкций `${var:-default}` из `bash`, но можно использовать `IF NOT DEFINED` для установки значений по умолчанию:

```cmd
IF NOT DEFINED name SET name=Guest
ECHO %name%  REM Выводит: Guest, если name не определена
```

## 8. Циклы

### 8.1 Цикл FOR

Цикл `FOR` используется для итерации по списку или диапазону чисел:

```cmd
FOR /L %%i IN (1,1,5) DO ECHO %%i
```

- `/L` указывает, что цикл работает с диапазоном чисел.
- Формат: `(start,step,end)` — начальное значение, шаг, конечное значение.

Пример перебора файлов:
```cmd
FOR %%f IN (*.txt) DO ECHO Обработка файла: %%f
```

> **Примечание**: В скриптах используйте `%%i` вместо `%i`, так как `%i` работает только в командной строке.

### 8.2 Цикл WHILE (с использованием IF и GOTO)

В `cmd` нет встроенного цикла `WHILE`, но его можно эмулировать с помощью `IF` и `GOTO`:

```cmd
SET count=0
:loop
IF %count% LSS 5 (
    ECHO %count%
    SET /A count+=1
    GOTO loop
)
```

## 9. Функции

Функции в `cmd` создаются с помощью меток (`:label`) и вызываются командой `CALL`. Они позволяют группировать команды для повторного использования.

### 9.1 Определение функции

```cmd
:my_function
ECHO Привет, %1!
GOTO :EOF
```

### 9.2 Вызов функции

```cmd
CALL :my_function Alice
```

Пример полной функции:
```cmd
@ECHO OFF
CALL :greet Alice
CALL :greet Bob
GOTO :EOF

:greet
ECHO Привет, %1!
GOTO :EOF
```

- `GOTO :EOF` завершает выполнение функции.
- Аргументы передаются через `%1`, `%2` и т.д.

> **Примечание**: Функции в `cmd` менее гибкие, чем в `bash/zsh`, и не поддерживают локальные переменные без дополнительных настроек.

## 10. Массивы (эмуляция)

`cmd` не поддерживает массивы напрямую, но их можно эмулировать, используя переменные с индексами:

```cmd
SET array_1=cat
SET array_2=dog
SET array_3=mouse
```

Пример перебора:
```cmd
SET i=1
:array_loop
IF DEFINED array_%i% (
    ECHO Элемент %i%: %array_%i%
    SET /A i+=1
    GOTO array_loop
)
```

## 11. Работа с файлами

### 11.1 Проверка существования файла

```cmd
IF EXIST file.txt (
    ECHO file.txt существует
) ELSE (
    ECHO file.txt не существует
)
```

### 11.2 Чтение из файла

```cmd
FOR /F "tokens=*" %%line IN (input.txt) DO ECHO %%line
```

### 11.3 Запись в файл

```cmd
ECHO Привет, мир! > output.txt
```

### 11.4 Добавление в файл

```cmd
ECHO Еще текст >> output.txt
```

## 12. Примеры

### 12.1 Простой счётчик

```cmd
@ECHO OFF
SET count=0
:loop
IF %count% LSS 5 (
    ECHO Счётчик: %count%
    SET /A count+=1
    GOTO loop
)
```

### 12.2 Приветствие с вводом пользователя

```cmd
@ECHO OFF
SET /P name=Введите ваше имя: 
IF NOT DEFINED name SET name=Guest
ECHO Привет, %name%!
```

### 12.3 Обработка файлов

```cmd
@ECHO OFF
FOR %%f IN (*.txt) DO (
    ECHO Обработка файла: %%f
    COPY %%f backup_%%f
)
```

---

## 13. Подсказки и рекомендации

- **Используйте `@ECHO OFF`**: Это скрывает команды от вывода в консоли, делая вывод чище.
- **Заключайте переменные в кавычки**: Используйте `"%var%"` в условиях, чтобы избежать ошибок с пустыми значениями.
- **Используйте `SET /A` для арифметики**: Это упрощает работу с числами.
- **Проверяйте существование переменных**: Используйте `IF DEFINED` для обработки неопределённых переменных.
- **Избегайте пробелов в именах файлов**: Это может вызвать ошибки при работе с командами, такими как `COPY`.
