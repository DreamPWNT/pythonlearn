### Режимы работы open
| Флаг  | Значение                                                        |
| ----- | --------------------------------------------------------------- | 
| `'r'` | Чтение. Файл должен существовать.                |
| `'w'` | Запись. Если файл существует – очищается.|
| `'x'` | Создание нового файла для записи. Ошибка, если файл существует. |
| `'a'` | Добавление в конец. Файл создаётся, если не существует.         |
| `'b'` | Двоичный режим (чтение/запись байтов, а не строк).              |
| `'t'` | Текстовый режим (по умолчанию). Чтение/запись строк.            |
| `'+'` | Открытие для чтения и записи одновременно.                      |

### Возможные комбинации
| Режим   | Поведение                                                                       |
| ------- | ------------------------------------------------------------------------------- |
| `'r'`   | Чтение текста. Ошибка, если файла нет.                                          |
| `'rb'`  | Чтение байтов. Ошибка, если файла нет.                                          |
| `'r+'`  | Чтение и запись текста. Файл должен существовать.                               |
| `'rb+'` | Чтение и запись байтов. Файл должен существовать.                               |
| `'w'`   | Запись текста. Содержимое стирается.                                            |
| `'wb'`  | Запись байтов. Содержимое стирается.                                            |
| `'w+'`  | Чтение и запись текста. Содержимое стирается.                                   |
| `'wb+'` | Чтение и запись байтов. Содержимое стирается.                                   |
| `'a'`   | Добавление текста в конец. Файл создаётся, если его нет.                        |
| `'ab'`  | Добавление байтов в конец. Файл создаётся, если его нет.                        |
| `'a+'`  | Чтение и добавление текста. Указатель в конце файла.                            |
| `'ab+'` | Чтение и добавление байтов. Указатель в конце файла.                            |
| `'x'`   | Создание нового текстового файла. Ошибка, если файл существует.                 |
| `'xb'`  | Создание нового бинарного файла. Ошибка, если файл существует.                  |
| `'x+'`  | Создание нового файла для чтения и записи текста. Ошибка, если файл существует. |
| `'xb+'` | Создание нового файла для чтения и записи байтов. Ошибка, если файл существует. |


## Другие параметры open
| Параметр        | Тип / Значения                                                                    | По умолчанию                                               | Когда используется     | Что делает                                                                                                                                                                                                                                                                                                                                                                   |
| --------------- | --------------------------------------------------------------------------------- | ---------------------------------------------------------- | ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`buffering`** | `-1` (по умолчанию), `0`, `1`, `>1`                                               | `-1` (автовыбор)                                           | Всегда                 | Управляет буферизацией ввода-вывода:<br>• `0` – без буфера (только в бинарном режиме)<br>• `1` – построчная буферизация (только текстовый режим)<br>• `>1` – размер буфера в байтах<br>• `-1` – выбор по умолчанию ОС                                                                                                                                                        |
| **`encoding`**  | строка (например `'utf-8'`, `'cp1251'`)                                           | `locale.getpreferredencoding(False)` (системная кодировка) | Только текстовый режим | Определяет кодировку при чтении/записи строк.<br>В бинарном режиме (`'b'`) нельзя указывать.                                                                                                                                                                                                                                                                                 |
| **`newline`**   | `None`, `''`, `'\n'`, `'\r'`, `'\r\n'`                                            | `None`                                                     | Только текстовый режим | Управляет преобразованием символов конца строки:<br>• `None` – перевод строк из файлов нормализуется к `\n` в Python, а при записи `\n` превращается в системный перевод (`\r\n` на Windows, `\n` на Unix).<br>• `''` – режим "универсального" чтения: сохраняются все варианты `\r`, `\n`, `\r\n`.<br>• `'\n'`, `'\r'`, `'\r\n'` – жёсткая фиксация формата перевода строк. |
| **`errors`**    | `'strict'`, `'ignore'`, `'replace'`, `'backslashreplace'`, `'namereplace'`, и др. | `'strict'`                                                 | Только текстовый режим | Задаёт стратегию обработки ошибок при кодировке/декодировке:<br>• `'strict'` – ошибка при несоответствии символа<br>• `'ignore'` – пропуск проблемных символов<br>• `'replace'` – замена на `?`<br>• `'backslashreplace'` – замена на `\uXXXX`<br>• `'namereplace'` – замена на `\N{...}`                                                                                    |

## Перевод строк в разных ОС

| Что в файле (байты) | Обозначение   | Как выглядит в Windows                                                           | Как выглядит в Linux/macOS                                                    | Как интерпретирует Python (`open(..., newline=None)`) |
| ------------------- | ------------- | -------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- | ----------------------------------------------------- |
| `0x0A`              | `\n` (LF)     | (старый Блокнот → всё в одну строку) <br> Современные редакторы → перенос строки | Перенос строки                                                                | Всегда `\n`                                           |
| `0x0D`              | `\r` (CR)     | В Блокноте и многих редакторах → всё в одну строку                               | В терминале может «стереть» начало строки (символ возврата каретки)           | Всегда `\n`                                           |
| `0x0D 0x0A`         | `\r\n` (CRLF) | Корректный перенос строки                                                        | Отображается как `строка^M` в `cat` / `vim`, либо как перенос + лишний символ | Всегда `\n`                                           |

### Пример одной и той же строки "Hello\nWorld"
| Содержимое файла   | Windows (старый Notepad) | Linux/macOS (`cat`)                    | Python чтение        |
| ------------------ | ------------------------ | -------------------------------------- | -------------------- |
| `"Hello\nWorld"`   | Всё в одной строке       | Две строки                             | `["Hello", "World"]` |
| `"Hello\rWorld"`   | Всё в одной строке       | `World` «перепишет» `Hello` при выводе | `["Hello", "World"]` |
| `"Hello\r\nWorld"` | Две строки (нормально)   | `Hello^M` + новая строка               | `["Hello", "World"]` |
